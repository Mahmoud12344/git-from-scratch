## what is a git object ?

- files in the git repository, whose paths are determined by their contents.

## what does the hash-object does

- it converts an existing file into a git object

## what does the cat-file does

- printing an exsiting hash obj to the standered output

## what is a tree:

- is a reference to a tree object, A tree maps blobs IDs to filesystem locations,
  and describes a state of the work tree.
  Put simply, it is the actual content of the commit: file contents, and where they go.

## what is a parent :/

> is a reference to the parent of this commit. It may be repeated: merge commits, for example, have multiple parents. It may also be absent: the very first  
> commit in a repository obviously doesn’t have a parent.
